WHAT HAPPENS WHEN...
--------------------

1. YOU STOP THE POD (to save money)
   ┌────────────────────────────────┐
   │ Container Disk: ❌ WIPED       │
   │ Volume Disk:    ✅ KEPT        │
   └────────────────────────────────┘
   
   Result: Start pod → Database & models still there
   Startup time: 2 minutes (just restart services)

2. YOU START THE POD (next morning)
   ┌────────────────────────────────┐
   │ Container Disk: Fresh Ubuntu   │
   │ Volume Disk:    Your data      │
   └────────────────────────────────┘
   
   Action: Run /workspace/start_legal_advisor.sh
   Result: Back in business!

3. YOU DELETE THE POD (⚠️ DANGER)
   ┌────────────────────────────────┐
   │ Container Disk: ❌ GONE        │
   │ Volume Disk:    ❌ GONE TOO!   │
   └────────────────────────────────┘
   
   ⚠️  MUST backup first!
   ./backup_runpod.sh <ip>

VERIFICATION COMMANDS

# Check workspace is on volume
mount | grep workspace

# Check database location
ls -la /workspace/nz_legal_rag/chroma_db/

# Check models on volume
ls -la /workspace/ollama_models/

# Check available space
du -sh /workspace/*
